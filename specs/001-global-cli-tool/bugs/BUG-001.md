# Bug Report: Package Not Found on PyPI

**Type**: Dependency issue
**Severity**: Critical
**Feature**: 001-global-cli-tool
**Reported**: 2026-06-26

## Description

Running `uv tool install otel-agent` fails with:

```
× No solution found when resolving dependencies:
╰─▶ Because otel-agent was not found in the package registry and you require
    otel-agent, we can conclude that your requirements are unsatisfiable.
```

The package has not been published to PyPI. Users cannot install the tool via any of the three specified methods (`uvx`, `uv tool install`, `pip install`).

## Artifact Traceability

### spec.md
- **Affected user story**: US1 — One-Command Install (P1)
- **Affected requirements**:
  - FR-001: Installable via `uvx otel-agent`
  - FR-002: Installable via `uv tool install otel-agent`
  - FR-003: Installable via `pip install otel-agent`
  - FR-011: PyPI package name MUST be `otel-agent`
- **Gap identified**: Spec defines installability requirements but has no requirement for "package MUST be published to a package registry." The spec assumes publication happens but doesn't mandate it as a prerequisite.

### plan.md
- **Affected sections**: Phase 3 — User Story 1: One-Command Install
- **Impact**: Plan has no task for publishing to PyPI. T011-T012 only verify local install, not registry publication.

### tasks.md
- **Affected tasks**:
  - T011 [US1]: Verify pyproject.toml has correct entry — ✅ done but insufficient
  - T012 [US1]: Test install with `uv pip install -e .` — ✅ done but only tests local editable install
- **False completions**: T012 is marked complete but only verified local install (`-e .`), not registry install (`uv tool install` from PyPI)
- **Missing tasks**:
  - T032: Build package with `python -m build`
  - T033: Publish to PyPI with `twine upload` or `uv publish`
  - T034: Verify `uvx otel-agent --version` works from PyPI

## Root Cause Analysis

The spec defined FR-001/FR-002/FR-003 as installability requirements but did not distinguish between "package is buildable" and "package is published to a registry." The plan mapped these to local-install verification tasks (T011-T012) without including a publication step. This is a spec gap — the implicit assumption that "buildable = installable" doesn't hold for remote registry installs.

## Recommended Fix

1. Add publication tasks to tasks.md
2. Build the package: `uv run python -m build`
3. Publish to PyPI: `uv run twine upload dist/*` (requires PyPI API token)
4. Verify: `uvx otel-agent --version`
5. Update spec to clarify installability means "available on PyPI"
