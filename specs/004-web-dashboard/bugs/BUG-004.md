# Bug Report: Noisy ConnectionResetError in Dashboard Console

**Type**: Implementation drift
**Severity**: Medium
**Feature**: 004-web-dashboard
**Reported**: 2026-06-27

## Description

The dashboard console is flooded with `ConnectionResetError: [Errno 54] Connection reset by peer` traceback messages. These occur when clients (browser favicon requests, SSE reconnects, premature disconnects) close the connection before the server finishes reading the request. This is normal HTTP behavior but the server logs full tracebacks for each occurrence.

## Root Cause Analysis

`ThreadingHTTPServer` inherits `handle_error()` from `socketserver.BaseServer`, which prints full tracebacks to stderr for any exception during request handling. `ConnectionResetError` is expected when clients disconnect early.

## Recommended Fix

Override `handle_error()` in `DashboardHandler` to suppress `ConnectionResetError` and `BrokenPipeError` tracebacks while still logging unexpected errors.
