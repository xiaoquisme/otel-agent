# Bug Report: Dashboard Cannot See History Requests

**Type**: Implementation drift
**Severity**: High
**Feature**: 004-web-dashboard
**Reported**: 2026-06-26

## Description

The dashboard shows "No requests logged yet" even though the proxy has been running and logging requests. The root cause is that the default database path `telemetry.db` is a relative path, which resolves to different files depending on the current working directory. If `otel-agent proxy` and `otel-agent dashboard` are started from different directories, they use different database files.

## Artifact Traceability

### spec.md
- **Affected user story**: US1 — Browse Request Logs (P1)

- **Gap identified**: Spec does not specify that the database path must be consistent across commands. The assumption "same machine" (Assumptions section) is insufficient — it should be "same database file."

## Root Cause Analysis

The default database path `telemetry.db` is a relative path. Both `otel-agent proxy` and `otel-agent dashboard` default to this relative path, but resolve it against different working directories. The config file already uses `~/.otel-agent/config.yaml` (absolute), but the database path was not updated to match.

## Recommended Fix

1. Change default database path from `telemetry.db` to `~/.otel-agent/telemetry.db` in all commands (proxy, dashboard, view)
2. Ensure `~/.otel-agent/` directory is created before writing to the database
3. Run `/speckit.bugfix.patch` to update spec artifacts
4. Run `/speckit.bugfix.verify` to confirm consistency
5. Resume `/speckit.implement` to apply the code fix
