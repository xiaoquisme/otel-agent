# Bug Report: Proxy startup crashes with AttributeError on config._providers

**Type**: Implementation drift
**Severity**: Critical
**Feature**: 007-multi-provider-routing
**Reported**: 2026-06-30
**Status**: Patched
**Patched**: 2026-06-30

## Description

Starting the proxy with the new provider-type config format (`providers:` with nested lists) crashes immediately with:

```
AttributeError: 'Config' object has no attribute '_providers'
```

The crash occurs in `src/otel_agent/commands/proxy.py` line 148, inside `_run_proxy()`. The Config class was refactored from a `_providers` dict (old model) to `_provider_types` / `_active_lookup` dicts (new model), but the proxy startup display code was not updated to match.

The proxy prints its listening message before crashing, so mitmproxy may briefly bind the port before the error kills the process.

## Steps to Reproduce

1. Configure `~/.otel-agent/config.yaml` with the new provider-type schema (providers.openai / providers.anthropic with lists of entries)
2. Run `otel-agent proxy start`
3. Proxy fails with AttributeError

## Root Cause Analysis

The Config class was refactored in Phase 2 (T003) from:
- `_providers: dict[str, Provider]` with `Provider.keys`, `Provider.active_keys()`

To:
- `_provider_types: dict[str, ProviderType]` with `ProviderType.entries`, `ProviderType.active_entry()`

The proxy startup display code at `proxy.py:148-151` still references the old model:
```python
for name, provider in config._providers.items():
    active = len(provider.active_keys())
    total = len(provider.keys)
    print(f"  provider: {name} ({active}/{total} keys active)")
```

This is implementation drift — the config was updated but the proxy display code wasn't.

## Recommended Fix

1. Update `proxy.py:148-151` to iterate `config.provider_types` instead of `config._providers`, using the new `ProviderType` and `ProviderEntry` model.
2. Run `/speckit.bugfix.patch` to update spec artifacts (plan.md, tasks.md)
3. Run `/speckit.bugfix.verify` to confirm consistency
4. Resume `/speckit.implement` to apply the code fix
