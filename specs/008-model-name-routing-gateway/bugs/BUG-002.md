# Bug Report: Tilde not expanded — creates literal `~` directory

**Type**: Implementation drift
**Severity**: Critical
**Feature**: 008-model-name-routing-gateway
**Reported**: 2026-07-07

## Description

When running `otel-agent proxy` (background mode), the `~` in default paths like `~/.otel-agent/config.yaml` is not expanded before being passed to the subprocess. This causes the subprocess to create a literal directory named `~` at the current working directory (e.g., `/Users/xiaoqu/~/.otel-agent/`) instead of using the user's home directory.

Two issues found:
1. `proxy.py:80-81` — `args.db` and `args.config` are passed raw (with `~`) to the subprocess command
2. `proxy.py:119` — `Path(args.db)` does not call `.expanduser()`, so the telemetry DB is written to the wrong location

## Root Cause Analysis

The CLI default values use `~/.otel-agent/...` as string literals (argparse doesn't expand them). The foreground runner (`_run_server`) expands the config path but not the DB path. The background launcher passes raw strings to `subprocess.Popen`, which doesn't invoke a shell to expand `~`.

## Recommended Fix

1. In `proxy.py` background launcher: expand `~` in subprocess command args
2. In `proxy.py` `_run_server`: expand `~` on DB path
3. Clean up the accidentally created `~/~/` directory
