# Bug Report: FastAPI startup crash — invalid response type annotation

**Type**: Implementation drift
**Severity**: Critical
**Feature**: 008-model-name-routing-gateway
**Reported**: 2026-07-07

## Description

Gateway fails to start with `FastAPIError: Invalid args for response field!` when calling `otel-agent proxy`. The error occurs because `@app.post("/v1/chat/completions")` declares a return type of `StreamingResponse | JSONResponse`, which FastAPI tries to interpret as a Pydantic model for OpenAPI schema generation. `StreamingResponse` and `JSONResponse` are Starlette response classes, not valid Pydantic field types.

## Stack Trace

```

is a valid Pydantic field type.
```

## Root Cause Analysis

Implementation error. The endpoint functions declare `-> StreamingResponse | JSONResponse` as return type annotations. FastAPI's OpenAPI schema generator interprets return annotations as Pydantic response models and fails because Starlette response classes are not Pydantic types. The fix is either to remove the return type annotations or add `response_model=None` to the route decorators.

## Recommended Fix

Add `response_model=None` to both endpoint decorators:

```python
@app.post("/v1/chat/completions", response_model=None)
@app.post("/v1/messages", response_model=None)
```
