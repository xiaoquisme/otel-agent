# Bug Report: Tool Message Content Rendered as Garbled Markdown

**Type**: Implementation drift
**Severity**: Medium
**Feature**: 018-fix-body-rendering
**Reported**: 2026-07-10
**Status**: Patched
**Patched**: 2026-07-10

## Description

Tool messages (role=tool) in request bodies have their content rendered through `renderMarkdown()` (marked.parse), which mangles JSON content since JSON is not valid markdown. The pretty-printed JSON output from `extractOpenAiRequestContent()` gets treated as markdown paragraphs, producing garbled/unreadable text in the formatted chat view.

For data ID 1117 (137 messages, 474KB body), tool messages contain JSON like `{"total_count": 1, "matches": [...]}` which should display as readable formatted JSON, but instead renders as broken markdown text.

**Root cause**: In `renderChatMessage()` (index.html:567), tool content passes through the `renderMarkdown()` fallback path because it doesn't contain `tool-call` HTML. The `renderMarkdown()` function uses `marked.parse()` which interprets JSON braces, brackets, and quotes as markdown syntax.

## Root Cause Analysis

This is an **implementation drift** — the code path for tool messages exists but routes through the wrong renderer. `extractOpenAiRequestContent()` correctly pretty-prints tool content as JSON (line 522-526), but `renderChatMessage()` passes it through `renderMarkdown()` (marked.parse) instead of rendering it as a `<pre>` code block.

The `isHtmlContent` check in `renderChatMessage` only detects assistant tool-call HTML (`<div class="tool-call">`), not tool message JSON content.

Alternatively, wrap tool content in `<pre>` tags in `extractOpenAiRequestContent()` before returning.
